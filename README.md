SOFTWARE REQUIREMENTS SPECIFICATION (SRS)
Project Title: Smart Student Time Management App
1. Introduction
The Smart Student Time Management App is an Android application designed to help students
manage their study time effectively using smartphone sensors. This document specifies the
functional and non-functional requirements of the system.
2. Purpose
The purpose of this document is to clearly define the system requirements for academic evaluation,
development, and future maintenance.
3. Scope
The application monitors student activity patterns using mobile sensors to classify time into
productive and unproductive periods and provides actionable feedback.
4. Overall Description
The system is a standalone Android application developed using Kotlin. It runs on Android devices
with built-in sensors and does not require external hardware.
5. Functional Requirements
1 User registration and profile management
2 Sensor-based activity monitoring
3 Study vs distraction time classification
4 Smart notifications and reminders
5 Daily and weekly productivity reports
6. Sensors Used
1 Accelerometer – detects movement and inactivity
2 Light Sensor – detects screen exposure
3 Proximity Sensor (optional) – detects phone usage position
7. Non-Functional Requirements
1 High performance with low battery usage
2 User-friendly interface
3 Secure handling of personal data
4 Offline functionality
8. Future Enhancements
Future versions may include machine learning-based productivity prediction, academic calendar
integration, and lecturer dashboards.
